Static Model:

文件数据包结构：
组成 :

如果一个xrp(xevol3d package)里包含了一个Static Model．
则这个Model有一个总目录，目录项目有cddt_static_model_dir的标记，
model_desc 文件，该文件为一个普通文件，但是有cddt_static_model_desc的标记

每个Mesh有一个目录，目录名字为记录在model_desc中Mesh名字列表相同
下表里记录了每个Mesh目录下的各个文件的作用

desc文件 详细内容见下
material 记录了这个文件用的material
position 位置（开头int记录了有几个顶点，每个顶点为一个sVector_t）
normal 如上
tangent TBN矩阵中的Tangent
diffuse diffuse颜色
uv_0-n 一共有n个uv_n，n个数记录在desc文件里

model_desc 文件的结构：
StaticModelDesc_t : 描述信息
          sBoundBox_t     : 这个模型的包围合
          <MeshName x N Mesh   const char m_MeshName[64]>

 

Mesh目录中的desc文件的结构:
      int nSubMesh:        一共几个subMesh
      sMeshDesc_t + N 个 sSubMeshDesc_t．一个变长的结构体
      sBoundBox_t : 包围合．这个Mesh的包围盒的大小  

Mesh目录中material文件的结构:
      int nMat:        一共n个Material
      <Material 信息 x nMat
            sModelMaterialHeader : 这个Material的一些总信息．比如有几个面等(和sSubMeshDesc_t中的数值一致)
            <TexturesModelTexLayer x N TextureLayer />
      /Material>
          

各个数据结构中的成员详细含义见成员名称或者注释

美术限制：
      1: 每个需要导出的Node应该有纹理坐标，有Material，
2: 应用到一个Mesh上的所有材质的UVMapChannel的使用情况要完全相同 //
